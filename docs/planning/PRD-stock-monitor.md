# PRD - 股票盯盘系统

## 1. Executive Summary

股票盯盘系统是一个个人量化工具，核心价值是：**发现交易信号 → 发出实时告警**。基于 Python Flask + SQLite + 原生前端的轻量级架构，支持策略组合扫描、实时盯盘、飞书通知。

## 2. Problem Statement

- 人工盯盘效率低，容易错过交易信号
- 主流量化平台（聚宽、米筐）太重，个人用户用不上
- 需要一个轻量级、自部署的盯盘工具

## 3. Objectives & KPIs

| 目标 | KPI | 验收标准 |
|------|-----|---------|
| 信号发现 | 策略覆盖率 | 支持 ≥5 种技术指标策略 |
| 实时性 | 告警延迟 | 信号触发后 ≤30 秒收到通知 |
| 易用性 | 页面加载时间 | 首屏 ≤2 秒（移动端） |
| 稳定性 | 可用性 | 交易时段 99% 可用 |

## 4. User Stories

**US-1: 自选股管理**
作为用户，我想添加/删除自选股，以便聚焦关注的标的。
- Given 我在股票页面，When 我搜索并点击"添加自选"，Then 股票出现在自选列表
- Given 我在自选列表，When 我点击删除，Then 股票从列表移除

**US-2: 策略组合扫描**
作为用户，我想组合多个策略同时扫描自选股，以便发现多维度信号。
- Given 我在策略页选中 2 个策略，When 点击"扫描自选股"，Then 返回符合所有策略的股票列表

**US-3: 实时告警**
作为用户，我想在策略触发时立即收到飞书通知，以便及时操作。
- Given 策略在交易时段触发，When 条件满足，Then 5 秒内推送飞书消息

**US-4: K 线查看**
作为用户，我想点击股票后查看 K 线图和基本面数据，以便做投资决策。
- Given 我点击某只股票，When 弹窗打开，Then 显示 K 线图 + 最新基本面数据

**US-5: 回测验证**
作为用户，我想对策略进行历史回测，以便验证策略有效性。
- Given 我选中策略和时间范围，When 运行回测，Then 返回收益率、胜率、最大回撤

## 5. Functional Requirements

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 总览 | 大盘指数、自选摘要、策略摘要、告警摘要 | P0 |
| 股票 | 自选股 CRUD、板块浏览、K 线弹窗、基本面数据 | P0 |
| 策略 | 策略库浏览、我的策略、组合扫描、回测、Walk-Forward | P1 |
| 告警 | 告警历史、筛选、标记已读、飞书通知 | P0 |

## 6. Non-Functional Requirements

- **性能**: 单次策略扫描 ≤5 秒（100 只股票）
- **安全**: 所有用户输入转义（XSS 防护）、SQL 参数化查询
- **可用性**: 移动端适配、PWA 支持
- **可维护性**: 前端模块化、后端 Blueprint 分层

## 7. Release Plan

- v1.0: 总览 + 股票 + 告警（当前状态）
- v1.1: 策略组合扫描优化 + 回测集成
- v1.2: Walk-Forward 分析 + 基本面筛选增强

## 8. Constraints

- SQLite 单机部署，不支持高并发
- 数据源依赖腾讯财经 API（免费但有频率限制）
- 飞书通知需要配置 App 凭证
