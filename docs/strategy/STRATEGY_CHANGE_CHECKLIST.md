# 策略变更影响清单

> 用途：每次策略变更后，按这个清单逐项检查，避免“代码改了，但文档 / 评估 / 同步链路没跟上”。

---

## 0. 先判断这次属于哪类变更

### A. 只改筛选条件
例如：
- RSI 阈值改了
- BB 阈值改了
- TOP300 改 TOP500
- 持有天数改了
- 增加 / 删除 KDJ、放量、弱市过滤

### B. 改上线标准 / 合格标准
例如：
- 正收益率门槛变了
- 夏普 / 索提诺 / 最大回撤标准变了
- 测试样本门槛变了

### C. 新策略上线 / 老策略下线
例如：
- 新增正式策略
- 某个策略从候选转为实盘
- 某个策略降级为历史/对照

### D. 新策略依赖新数据
例如：
- 开始依赖北向 / 两融 / 股东 / 新闻 / 筹码 / 龙虎榜 / 大宗 / 行业 / 资金流

---

## 1. 生产入口检查（必查）

文件：`scripts/daily/daily_pick_combined.py`

检查项：
- [ ] 策略筛选条件已同步修改
- [ ] 持有规则 / TOPN / 弱市阈值已同步修改
- [ ] 输出结果字段已同步修改
- [ ] 推送文案与实际策略一致
- [ ] `load_strategy_metrics()` 读取的结果文件仍然正确
- [ ] `load_strategy_qualified()` 的判断逻辑仍然正确
- [ ] 新增策略时，已接入正式策略池 / 历史策略池
- [ ] 下线策略时，已从正式推送中移除

---

## 2. 数据依赖检查（按需必查）

文件：`scripts/daily/daily_sync.py`
文件：`scripts/daily/update_tencent.py`
文件：`scripts/daily/daily_pick_combined.py`

如果策略依赖新增数据，检查：
- [ ] 该数据是否已经有同步逻辑
- [ ] 是否在盘后自动任务中会提前同步
- [ ] 是否在选股前补齐流程中会自动兜底同步
- [ ] 是否在选股前校验中有 freshness / coverage 检查
- [ ] 表名 / 字段名 / 日期字段是否和生产代码一致

常见依赖：
- 北向资金 / 北向持股
- 融资融券
- 股东数据
- 新闻
- 复盘
- 筹码分布
- 涨跌停
- 指数K线
- 龙虎榜
- 大宗交易
- 资金流
- 行业板块

---

## 3. 评估层检查（只要策略规则变了就查）

文件：`docs/EVAL_FRAMEWORK.md`
文件夹：`scripts/evaluation/`

检查项：
- [ ] 这次变更是否需要重新评估
- [ ] 评估脚本是否与当前策略规则一致
- [ ] 评估结果文件路径是否变化
- [ ] 生产代码读取的评估结果文件名是否仍正确
- [ ] 上线标准若有变化，文档和代码是否一起改了

典型高风险点：
- 评估脚本改了，但 `daily_pick_combined.py` 还在读旧结果文件
- `EVAL_FRAMEWORK.md` 改了，但生产里的合格判断没改

---

## 4. 策略文档检查（必查）

### 4.1 总览文档
文件：`docs/strategy/STRATEGY_INDEX.md`

检查项：
- [ ] 当前实盘策略列表是否正确
- [ ] 状态（运行中 / 候选 / 历史 / 对照）是否正确
- [ ] 策略条件摘要是否正确
- [ ] 持有规则是否正确
- [ ] 上次评估日期是否正确
- [ ] 更新记录是否补上

### 4.2 单策略文档
文件：`docs/strategy/<策略名>/README.md`

检查项：
- [ ] 入场条件已更新
- [ ] 过滤条件已更新
- [ ] 持有规则已更新
- [ ] 风险提示已更新
- [ ] 与生产入口逻辑一致

---

## 5. 运行链路检查（按需必查）

文件：`docs/ops/crontab.md`
系统：真实 `crontab -l`

检查项：
- [ ] 文档中的时间与真实 crontab 一致
- [ ] 若新增数据依赖，盘后同步时序是否足够
- [ ] 17:00 / 17:30 / 20:30 的链路是否仍合理
- [ ] 兜底补齐逻辑是否仍适用
- [ ] 日志路径是否仍正确

---

## 6. 结果真源检查（强制建议）

每次策略变更后，至少确认一次：
- [ ] “策略文案” 和 “实际筛选条件” 一致
- [ ] “评估报告” 和 “生产读取结果” 一致
- [ ] “推送内容” 和 “当前正式策略池” 一致

特别注意：
- 不要只改文档不改代码
- 不要只改探索脚本不改生产入口
- 不要只改评估结果文件名不改读取逻辑

---

## 7. 最小检查模板（快速版）

如果只是一次普通策略变更，至少检查这 4 个地方：

- [ ] `scripts/daily/daily_pick_combined.py`
- [ ] `docs/strategy/STRATEGY_INDEX.md`
- [ ] 对应策略 `README.md`
- [ ] 如涉及新数据：`daily_sync.py`

---

## 8. 每次变更后的固定动作

每次策略变更完成后，必须执行：

1. **重新阅读本清单**
2. **逐项自查是否漏改文件**
3. **确认文档、评估、生产口径一致**
4. **如有 git 提交，提交前再次检查本清单**

---

## 9. 当前项目默认规则

当前项目里，**只要发生策略变更，我都默认要检查本文件。**

也就是说：
- 改策略条件 → 检查
- 改策略池 → 检查
- 改评估标准 → 检查
- 改数据依赖 → 检查
- 改推送口径 → 检查

这是固定动作，不再省略。
